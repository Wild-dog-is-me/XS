# 技术名词表

## 写作原则
技术名词出现时，要让非专业读者能通过上下文理解，让专业读者不觉得离谱。避免连续抛出五个以上术语。

| 名词 | 简短解释 | 写作使用方式 | 注意事项 |
|---|---|---|---|
| Transformer | 基于注意力机制的神经网络架构，是大模型基础 | 面试、技术争论、路线判断 | 2018年可前瞻，但行业认知尚未统一 |
| BERT | 2018年发布的预训练语言模型，代表NLP范式转变 | 作为主角判断预训练路线的证据 | 不要夸成通用大模型 |
| GPT | 生成式预训练模型路线 | 中后期作为行业爆点 | 2018早期不能让国内所有人都重视 |
| MLOps | 机器学习工程化、部署、监控和迭代体系 | 日升早期产品切入点 | 适合程序员创业质感 |
| RAG | 检索增强生成，把外部知识检索后交给模型回答 | 2023后企业落地关键 | 早期可埋概念，不能提前成熟化 |
| Agent | 能调用工具、规划任务并执行的软件智能体 | 中后期产品主线 | 要写清权限、审计、失败恢复 |
| 推理优化 | 降低模型服务延迟和成本的工程技术 | 主角专业优势 | 可通过客户压测、线上故障写戏 |
| GPU集群 | 多卡、多机组成的训练/推理基础设施 | 资金与资源主线 | 注意供应周期、机房、电力和网络 |

## 术语落地方式
1. 用对话讲立场。
2. 用故障讲原理。
3. 用客户需求讲价值。
4. 用成本账讲商业化。
